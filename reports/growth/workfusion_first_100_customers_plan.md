# Workfusion First 100 Customers Plan

## Separation Rule

Workfusionapp is the commercial product.

BoltIQ is the internal trading operations / future hedge-fund infrastructure track.

Do not mix the two:

- do not pitch BoltIQ partners as Workfusion customers unless they explicitly opt in;
- do not share BoltIQ telemetry, partner briefs, trading reports, broker information, or internal hedge-fund plans in Workfusion outreach;
- do not describe Workfusion as a trading system, signal provider, prop-firm pass service, or investment product;
- use Workfusion positioning only: AI EA Generator + Debugger for MT4/MT5 traders.

## Offer

Core offer:

> Workfusion helps MT4/MT5 traders and MQL developers turn strategy ideas into EA drafts, debug compiler errors, risk-check the code, organize projects, and download MQL outputs.

What we sell:

- faster first EA draft;
- less time lost on MetaEditor compile errors;
- cleaner handoff to a real MQL developer;
- risk/readiness review before manual backtesting;
- project organization for EA builds.

What we never sell:

- guaranteed profits;
- prop challenge pass guarantees;
- live trading execution;
- account management;
- broker/API credential access;
- investment advice.

## Ideal First Customers

1. MQL5 developers who receive messy EA requests and want cleaner first drafts.
2. MT4 traders trying to fix old EAs or migrate logic to MT5.
3. Prop-style traders who need risk/readiness checks before manual testing.
4. Small EA sellers who need support triage and compiler-error reduction.
5. Trading educators who teach automation and need a practical workflow tool.

## Qualification Questions

Ask only after someone opts in or directly asks about the product:

1. Are you building MT4 or MT5?
2. Are you starting from a strategy idea, existing code, or compiler errors?
3. What exact error or bottleneck is blocking you now?
4. Do you want generation, debugging, code review, risk/readiness scoring, or compile support?
5. Are you a trader, MQL developer, freelancer, educator, or EA seller?
6. Would saving 2-5 hours per EA build justify a paid subscription?

## First Message Templates

### Helpful forum reply

Short answer: this looks like a MetaEditor / EA-structure issue, not a trading-performance issue.

I would check:

1. whether the first compiler error is causing the later cascade;
2. whether the EA has a complete `OnInit` / `OnTick` lifecycle;
3. whether trade calls, spread checks, stops, and lot sizing are handled explicitly.

If useful, I built Workfusion for this exact EA workflow: paste the idea/code/error, generate a corrected MQL draft, then run a risk/readiness check before manual testing.

### LinkedIn build note

I am building Workfusionapp for MT4/MT5 EA builders.

The current workflow is simple:

1. describe an EA idea;
2. generate a complete MQL draft;
3. paste MetaEditor compiler errors;
4. get a corrected draft;
5. risk-check the EA before manual backtesting;
6. download the MQL output.

It is not a signal service and does not promise trading performance. It is a development workflow for people who build, debug, and test Expert Advisors.

Looking for serious MQL4/MQL5 builders who want to test it and send direct feedback.

### Partner message to MQL freelancer

I am building Workfusionapp, an AI EA Generator + Debugger for MT4/MT5 traders.

I do not see this as replacing MQL developers. The useful angle is earlier in the workflow: cleaner first drafts, compiler-error summaries, risk/readiness notes, and better project context before a real developer does the serious review.

Would you be open to testing whether it reduces repetitive cleanup work for EA requests?

## Decision Gates

After 14 days:

- if people use generation but do not pay, improve pricing/onboarding;
- if people ask mostly for compiler fixes, make compiler workflow the main CTA;
- if people ask for strategy profitability, tighten copy to avoid wrong-fit users;
- if MQL freelancers respond well, build an affiliate/partner track;
- if support is mostly confusion, rewrite the first screen and onboarding.
